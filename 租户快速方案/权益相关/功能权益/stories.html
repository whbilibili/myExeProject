典型用户故事

角色：平台运营方（产品经理）

任务：设计新的订阅计划

背景：市场反馈需要一个介于专业版和企业版之间的计划

操作流程：

  1. 登录平台管理后台 → 订阅计划管理
  2. 点击"创建新计划"，命名为"团队版"
  3. 配置功能包：启用"AI工作流节点"，禁用"企业级插件开发"
  4. 设置资源配额：50个智能体、100GB存储、1000万API调用/月
  5. 定价：299元/月/租户
  6. 发布计划，对所有新注册租户可见
新的"团队版"计划上线，为中型客户提供了更合适的选择

角色：租户管理员（李总）

任务：为研发团队开通AI功能

背景：公司刚订阅了"专业版"计划，包含AI分析功能和10个相关席位

操作流程：

  1. 登录租户管理后台 → 功能权限分配
  2. 选择"AI工作流节点"功能（显示：已分配 0/10 席位）
  3. 点击"管理分配"，打开成员选择器
  4. 从左侧"可选成员"中，搜索并选择研发团队的7名工程师
  5. 点击"→"将他们移动到右侧"已分配成员"
  6. 保存，系统显示：已分配 7/10 席位
这7名工程师在任何他们有编辑权限的空间中，都能使用AI节点

角色：空间管理员（王项目经理）

任务：为跨部门项目分配协作权限

背景："客服升级项目"空间需要邀请市场部同事协作

操作流程：

  1. 进入"客服升级项目"空间 → 成员管理
  2. 点击"添加协作者"
  3. 搜索并选择市场部的张三、李四
  4. 为张三分配"可编辑"角色（需要他修改智能体话术）
  5. 为李四分配"查看与使用"角色（只需要测试功能）
  6. 保存设置
张三可以编辑项目内的智能体，李四只能查看和测试。若张三被租户管理员授予了AI功能权限，他还能使用AI节点优化智能体

角色：普通用户（张工程师）

任务：使用AI功能优化工作流

背景：张工程师被授予了AI功能，正在"数据分析项目"中创建工作流

操作流程：

  1. 进入工作流编排界面
  2. 在节点面板中，看到"AI分析"分类（因为租户订阅了支持AI的计划 + 被分配了功能权限）
  3. 拖拽"智能数据洞察"节点到画布
  4. 配置节点参数，连接数据源
  5. 测试运行，获得AI生成的分析报告
如果张工程师没有AI功能权限，第2步时他看不到"AI分析"分类；如果租户订阅的是基础版，整个AI功能都不会出现
