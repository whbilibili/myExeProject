# OpenAI风格设计指南

> 本项目采用OpenAI风格的UI设计，强调简洁、紧凑和高可读性

## 🎯 设计原则

### 1. 紧凑但不拥挤
- 减小字体大小，但保持足够的行高和间距
- 使用更小的图标和组件，但保持清晰可点击

### 2. 清晰的视觉层次
- 通过字重和颜色区分信息层级
- 避免过度使用大字体造成的视觉混乱

### 3. 一致性
- 所有文字类内容最大14px
- 数值类内容可以稍大以突出重要信息

## 📏 字体规范

### 文字类字体

| 用途 | 字号 | 字重 | 使用场景 |
|------|------|------|----------|
| 页面标题 | 14px | 600 (Semibold) | 导航栏标题、卡片标题、区块标题 |
| 正文内容 | 13px | 400 (Regular) | 正文、说明文字、表单输入 |
| 辅助信息 | 12px | 500 (Medium) | 标签、提示、次要信息 |
| 微小文字 | 11px | 400 (Regular) | 时间戳、版权信息 |

### 数值类字体

| 用途 | 字号 | 字重 | 使用场景 |
|------|------|------|----------|
| 主要数值 | 24px | 600 (Semibold) | 统计卡片主数值 |
| 次要数值 | 20px | 600 (Semibold) | 指标卡片数值 |
| 一般数值 | 14px | 500 (Medium) | 表格中的数字 |

## 🎨 组件规范

### 导航栏
```css
height: 56px;           /* 比标准64px更紧凑 */
font-size: 14px;        /* 标题字号 */
icon-size: 20px;        /* 图标大小 */
```

### 卡片
```css
title-size: 14px;       /* 卡片标题 */
content-size: 13px;     /* 卡片内容 */
padding: 24px;          /* 内边距 */
border-radius: 8px;     /* 圆角 */
```

### 按钮
```css
font-size: 13px;        /* 按钮文字 */
height: 32px;           /* 默认按钮高度 */
padding: 0 16px;        /* 水平内边距 */
```

### 表格
```css
header-size: 13px;      /* 表头 */
cell-size: 13px;        /* 单元格 */
row-height: 48px;       /* 行高 */
```

### 表单
```css
label-size: 13px;       /* 标签 */
input-size: 13px;       /* 输入框 */
help-text: 12px;        /* 帮助文字 */
```

### 图标
```css
small-icon: 16px;       /* 小图标（行内） */
medium-icon: 20px;      /* 中等图标（导航栏） */
large-icon: 40px;       /* 大图标（配额卡片） */
```

## 🎨 颜色使用

### 文字颜色
```css
--text-1: #1D2129       /* 主要文字（标题） */

--text-3: #86909C       /* 辅助文字（说明） */
```

```css

--warning-color: #FF7D00    /* 警告 */

```

## 📐 间距系统

基于4px的间距系统：

```css
--spacing-1: 4px        /* 最小间距 */
--spacing-2: 8px        /* 小间距 */
--spacing-3: 12px       /* 中小间距 */
--spacing-4: 16px       /* 常规间距 */
--spacing-5: 20px       /* 中等间距 */
--spacing-6: 24px       /* 大间距 */
--spacing-8: 32px       /* 超大间距 */
```

## ✅ 设计检查清单

在设计新组件时，请确保：

- [ ] 所有文字类内容不超过14px
- [ ] 正文内容使用13px
- [ ] 辅助信息使用12px
- [ ] 数值类可以使用20-24px突出显示
- [ ] 图标大小合理（16-40px范围）
- [ ] 组件间距使用4px的倍数
- [ ] 保持清晰的视觉层次
- [ ] 按钮、输入框等可交互元素足够大（至少32px高）

## 💡 设计建议

### DO（推荐）
- ✅ 使用字重来区分层级
- ✅ 使用颜色来传达状态
- ✅ 保持充足的行高（1.5）
- ✅ 数值加粗突出显示
- ✅ 重要操作使用主色

### DON'T（避免）
- ❌ 不要使用超过14px的文字
- ❌ 不要过度使用粗体
- ❌ 不要使用过小的行高
- ❌ 不要让按钮太小（小于32px）
- ❌ 不要使用太多层级的字号

## 📱 响应式考虑

在移动端：
- 保持相同的字体大小（不要缩小）
- 增加按钮和可点击区域的大小
- 适当增加间距以适应触摸操作
- 简化布局，突出核心信息
