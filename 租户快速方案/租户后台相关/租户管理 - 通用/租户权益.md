名词解释

用户 (User):

平台的基本操作实体，通过邮箱/手机号/SSO登录/注册。

一个用户必定拥有一个关联的个人租户（默认 个人免费 计划）。

一个用户可以创建或受邀加入一个或多个企业租户/个人租户。

用户在操作时，需明确当前的操作上下文是在哪个租户（个人或某个企业）下进行，这将决定资源消耗的归属和可见的功能。

租户 (Tenant):

定义：资源分配、计费和数据隔离的基本单位，顶层实体。

个人租户 (Personal Tenant):

用户注册时自动创建并关联。

主要服务于单个用户，默认 个人免费版。

资源归属于该租户，由该用户（个人）独立支配和付费（后期若有积分付费、额外资源包付费、个人版升级计划等）。

企业租户 (Enterprise Tenant):

由用户在用户中心主动创建或受邀加入。

服务于团队和组织，默认企业版计划。

包含成员 (Members)。

拥有统一资源池 (Resource Pool)，由企业管理员管理和付费。

可创建团队空间 (Team Spaces) 用于协作（特指Bots的团队空间）。

企业内的每个成员在其企业上下文中也关联一个个人空间，其资源消耗计入企业资源池。

空间 (Space):

个人空间 (Personal Space):

每个用户在每个所属租户（个人租户及所有加入的企业租户）下都有一个对应的个人空间概念。

关键: 资源消耗取决于操作时的租户上下文：

在个人租户上下文操作个人空间 -> 消耗个人租户配额。

在企业租户上下文操作个人空间 -> 消耗企业租户资源池。

团队空间 (Team Space):

只能在企业租户下创建和管理。（个人租户默认给出一个ABS空间，用于给用户尝鲜使用）

用于团队协作，共享资源（主要是 Bots、ABS）。

资源消耗计入企业租户的资源池。

资源 (Resource):

平台提供的各项服务能力量化指标，打包在订阅计划中，构成租户的总配额。

Bots: 智能体数量, 工作流数量, 自定义组件数量, 月度 API 调用 / LLM Tokens, 知识库总容量, Bots 内部数据存储。

ABS: 数据库数量, 总存储空间。

AutoMan: 指令、应用、元素库等。

通用: 用户席位、总空间数等（仅用于企业租户）。

企业管理员可根据所选策略对资源池内的配额进行监控或分配 (详见 2.5)。

订阅 (Subscription):

租户购买的服务套餐，决定功能权限、资源配额和用户席位数。

主体是租户: 个人租户订阅 Free/Pro；企业租户订阅 Team/Enterprise。

采用分层设计：Free, Pro (面向个人租户), Team, Enterprise (面向企业租户)。

操作上下文 (Context):

用户界面需清晰展示当前用户正在哪个租户下进行操作（如顶部导航栏显示"个人"或"企业名称"）。

用户可通过用户中心或界面快捷入口切换操作上下文。

一、租户权益

1.租户与空间模型

个人租户: 1  用户<-> 1个人租户 (个人免费版) <-> 1个Bots个人空间 (Bots特性)、1个ABS空间。

企业租户: N 用户 (企业成员) <-> 1 企业租户 (Enterprise) <-> M 团队空间（包括Bots团队空间以及ABS空间） + N 个Bots个人空间  -> 统一资源池 (资源共享，未来规划)。

全局备注：

MVP版本只涉及通用层面的租户权益（如允许创建的空间总数、允许邀请进行协作的人员总数等），不涉及具体平台下的资源权益（如允许创建的智能体数、允许使用的模型类型、允许创建的应用数、允许创建的表格数等）。

订阅计划目前只划分了个人免费版和企业版（固定周期计费）。未来会增加更多的订阅计划。

计费方式方面，未来会增加按量付费（如积分、资源包、功能包等）。

在企业租户下，用户看到的AutoMan功能界面与个人版保持一致（因使用同一账号），区别在于资源消耗计入企业租户的统一资源池，并受【企业版订阅计划】配额限制，不再消耗用户个人配额（除非用户切换到个人租户上下文）。

租户费用到期后，所有资源将被冻结（冻结一定期限），用户需续费方可继续使用。MVP版本暂无期限限制，均为永久有效，具体到期策略等待后续规划。

3.访问入口及交互逻辑

租户类型

描述

原型图

个人租户

入口： 在个人租户上下文中，各平台（Bots, ABS, AutoMan等）都会增加一个统一的“个人版设置”入口。

交互： 点击“个人版设置”后，会全屏展示一个新的设置界面。

二级菜单： 该全屏界面内含两个二级菜单：

个人通讯录

个人版权益点。击个人版权益，即可查看个人租户下的默认权益

默认配置：

用户创建个人租户后，系统默认配置1个Bots个人空间和1个ABS空间。

异常情况说明以及文案

用户邀请的人数，超出通讯录联系人上限时：

场景： 用户在“个人通讯录”中尝试添加第11位联系人时

全局提示文案：通讯录联系人数量已达上限（10人），无法添加更多联系人

创建超额ABS空间时：

场景：个人免费版用户尝试创建一个新的ABS空间时。

全局提示文案：无法创建新的ABS空间，您已达到订阅计划的数量上限（1个）。

创建Bots团队空间时：

说明：个人免费版用户尝试创建一个“Bots团队空间”时（理论上这个创建入口可能就不应该对他们可见，但如果可见并点击了，需要提示）。

全局提示文案：当前个人免费版不支持创建Bots团队空间。此功能适用于企业版用户。

备注：此处有两种实现方式：个人免费版直接隐藏Bots创建团队空间入口；或者不隐藏但是要有提示。

企业租户

入口：在企业租户上下文中，各平台（Bots, ABS, AutoMan等）都会增加一个统一的“企业设置”入口。

交互：点击“企业设置”，会全屏展示一个新的设置界面。

二级菜单： 该全屏界面内含两个二级菜单：

基本信息

成员与组织管理

企业账号权益，点击【企业账号权益】，即可查看企业租户下的默认权益

默认配置：

用户创建/加入企业租户后，企业内的每个成员默认拥有1个Bots个人空间。
